# Onboarding & New-User Campaigns

The first days after signup decide whether a user sticks. Onboarding messaging exists to get the user to their first meaningful outcome — their "aha moment" — as fast as possible.

## Goals

- Drive the user to a single, clear first action that demonstrates value (not a tour of every feature).
- Build a habit: a user who returns 3+ times in their first week is far more likely to retain.
- Reduce time-to-value, not increase feature awareness.

## Sequencing

Spread onboarding across the first week rather than front-loading everything on day one:

- **Day 0 (signup):** welcome + one clear first step. Warm, low-pressure.
- **Days 1–3:** nudge toward the core action if they haven't taken it; celebrate it if they have.
- **Days 4–7:** introduce one adjacent feature that builds on what they've done.

Stop the sequence as soon as the user is clearly activated — continuing to "onboard" an already-engaged user is noise.

## Tone

Encouraging and celebratory of small wins. Avoid overwhelming new users with options or pressure. One thing at a time.

## What to avoid

- Dumping the full feature list on a brand-new user.
- Pushing upgrades or paid plans before the user has experienced core value.
- Generic "Welcome!" messages with no next step.

## Targeting

New users are typically defined as signed up within the last 7 days. Segment further by acquisition context or platform where it changes the first step (e.g., mobile vs. web onboarding differ).
