# Messaging Guidelines

This folder is Plotline's internal messaging best-practices knowledge base. The Campaign Copilot retrieves from these documents (RAG) to ground its segment and message recommendations in our standards, rather than inventing advice.

## Contents

- `01-brand-voice-and-tone.md` — how our messages should sound
- `02-re-engagement-playbook.md` — bringing lapsed users back
- `03-push-notification-copy.md` — writing effective push
- `04-channel-and-timing.md` — choosing channel and send time
- `05-segmentation-and-personalization.md` — building precise segments
- `06-onboarding-new-users.md` — activating new signups
- `07-winback-churned-users.md` — recovering churned users
- `08-feature-adoption-campaigns.md` — driving adoption of unused features
- `09-email-best-practices.md` — subject lines, deliverability, body
- `10-in-app-messaging.md` — formats and timing for in-app
- `11-ab-testing-and-experimentation.md` — testing messaging changes
- `12-frequency-capping-and-fatigue.md` — how often to message
- `13-incentives-and-offers.md` — when and how to use incentives
- `14-measuring-campaign-success.md` — picking the right success metric
- `15-consent-compliance-and-opt-outs.md` — permission and suppression
- `16-localization-and-timezones.md` — local language and send times
- `17-lifecycle-stages-glossary.md` — shared vocabulary for user states

Documents overlap and cross-reference intentionally — for a given campaign goal, only some are relevant, so retrieval quality matters.
